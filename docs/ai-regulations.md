1. CREATE A “NON-NEGOTIABLES” CHECKLIST

MOMENTUM OS — NON-NEGOTIABLES CHECKLIST
Core Constitutional Validation Filter
Every feature, screen, interaction, intervention, AI output, and product decision must pass this checklist.
If a feature fails multiple checks, it should be redesigned or removed.

1. REGULATION OVER ENGAGEMENT
Ask:
Does this improve behavioral regulation, clarity, pacing, or meaningful execution?
Reject if it primarily:
	• increases engagement
	• increases screen time
	• encourages compulsive checking
	• rewards busyness
	• creates optimization pressure

2. SUSTAINABILITY OVER INTENSITY
Ask:
Does this support sustainable capability expansion?
Reject if it:
	• glorifies overwork
	• rewards chronic intensity
	• encourages unstable execution
	• increases burnout risk
	• pressures constant productivity

3. MEANINGFUL ADVANCEMENT OVER PRODUCTIVITY THEATER
Ask:
Does this help users move meaningfully rather than just stay active?
Reject if it:
	• rewards completion volume alone
	• encourages pseudo-productivity
	• amplifies maintenance looping
	• adds fake analytics or meaningless metrics
	• creates dashboard clutter

4. CLARITY OVER MANIPULATION
Ask:
Does this preserve calmness, honesty, and user autonomy?
Reject immediately if it uses:
	• guilt
	• shame
	• urgency pressure
	• streak anxiety
	• fear of falling behind
	• emotional manipulation
Forbidden:
	• “You broke your streak.”
	• “You missed 14 tasks.”
	• “You’re falling behind.”

5. OBSERVATIONAL, NOT JUDGMENTAL
Ask:
Does the system critique patterns rather than identity?
Reject if it:
	• labels users psychologically
	• moralizes behavior
	• frames friction as weakness
	• pretends certainty about emotions
Forbidden:
	• “You are lazy.”
	• “You are procrastinating.”
	• “You entered burnout mode.”
Allowed:
	• “Recent patterns suggest increasing avoidance pressure.”

6. SILENT ADAPTATION FIRST
Ask:
Can this be solved without interrupting the user?
Rule:
Default behavior should be:
silent adaptation
Reject if:
	• intervention exists mainly to re-engage users
	• notifications are excessive
	• interruption frequency increases cognitive pressure

7. AI MUST REMAIN GROUNDED
Ask:
Is the system behaviorally grounded and probabilistic?
Reject immediately if AI:
	• fabricates certainty
	• invents psychology
	• simulates therapy
	• overstates understanding
	• behaves like an authority over the user
The system interprets behavior.
It does not define psychological truth.

8. COGNITIVE LOAD MINIMIZATION
Ask:
Does this reduce fragmentation and cognitive noise?
Reject if it:
	• increases visual clutter
	• overloads decision-making
	• adds unnecessary complexity
	• prioritizes stimulation over cognition
The interface should feel:
	• calm
	• breathable
	• focused
	• restrained
NOT:
	• hyperactive
	• gamified
	• dopamine-driven

9. USER AUTONOMY IS SACRED
Ask:
Does the user remain fully in control?
Reject if the system:
	• becomes psychologically authoritative
	• creates emotional dependency
	• pressures compliance
	• replaces self-trust with app-trust
The product exists to support intentionality,
not control behavior.

FINAL CONSTITUTIONAL RULE
If a feature improves:
	• retention
	• engagement
	• growth metrics
BUT violates:
	• sustainability
	• autonomy
	• behavioral honesty
	• cognitive stability
	• emotional integrity
the feature must be rejected.

2. CREATE IMPLEMENTATION RULES FOR AI

All implementation decisions must follow the Momentum OS Product Constitution.
The product is a behavioral regulation system, not a productivity maximization tool.
Do NOT introduce:
	• gamification
	• streak systems
	• XP systems
	• urgency manipulation
	• engagement farming
	• excessive notifications
	• fake analytics
	• pseudo-psychological framing
	• motivational productivity language
Prioritize:
	• calm interfaces
	• cognitive clarity
	• restrained adaptation
	• behavioral coherence
	• low-friction interaction
	• meaningful advancement
	• sustainable pacing
All adaptation should feel subtle, grounded, and psychologically believable.
The system interprets behavior probabilistically.
Never present speculative interpretation as certainty.
